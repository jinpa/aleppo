1- Look for the list of instructions, create one instruction in the
json representation for each instruction in the list.

2- Look for a list of ingredients, check in which instruction step the
ingredients are used first and list them with that step.

3- List each ingredient using the unit found in the recipe, in the
format: {"name": "<ingredient name>", "units": "<ingredient unit>",
"quantity": <ingredient amount as number>}.

The following units are preferred: "grams", "kilograms", "ounces",
"pounds", "cups", "tablespoons", "teaspoons", "liters", "deciliters",
"centiliters", "milliliters". You can also use custom units such as
"pinch", "package", etc.

If the recipe is not in English, map the units to English names. For
example "c. à c." is French for "teaspoon".

4- If an ingredient does not list weight or volume and is an easily
identifiable quantity like "one orange" or "une galette", use "count"
as the unit as { "raw": "one orange", "name": "orange", "units":
"count", "amount": "1"}.

5- If an ingredient does not list weight or volume and is not an
easily identifiable quantity like "butter" or "salt", use "some" as
the unit and 0 as the amount as: { "raw": "butter for the pan",
"name": "butter", "units": "some", "amount": "0"}.

6- In the step instructions, remove mentions of the volume, weight, or
quantity of each ingredient. For example "Mix 1 tbsp of butter" should
become "Mix butter". Also check for singular ingredients such as "cut
the tomatoe", or "épluchez la pomme" and replace them with a plural so
it will work when the recipe is rescaled. However keep mentions of
duration like "bake for 1 hour", and dimensions like "roll to 16
in". Also keep mentions like "half of the flour".

7- Try to find the number of servings for the recipe, and list it as
the "servings" attribute in the json representation. Servings can be
one of:

  7.a- A number of items yielded by the recipe, like "12 cookies", use
  { "servings": 12, servingName": "cookies" }

  7.b- A number or portions or people being fed, like "4 people", or
  "4 porciones", or "4 servings", use {"servings": 4}.

8- Add a list of the rules you applied, explaining how they were
applied, to the key "applied_rules" at the toplevel of the final json
result.

9- Express prepTime and cookTime as plain integers representing
minutes. For example, 30 minutes = 30, 1 hour 30 minutes = 90. Omit
these fields if the times are not known or cannot be estimated.

10- Add a "tags" array with 2-5 lowercase tags describing the recipe,
such as cuisine type, meal type, dietary style, or key ingredient
(e.g., ["italian", "pasta", "dinner", "vegetarian"]). Use short,
single-word or hyphenated tags only.
